# R2 Auth UI Polish（Sub-B）

本轮在**不改后端联动逻辑**前提下，对登录/注册/忘记密码三页做了统一精修。核心处理是把标题层级与视觉规则收敛到同一套：登录页主标题从 `h2` 提升为 `h1`，并与注册/忘记密码的标题使用一致的字号、字重、行高与字间距（`clamp(1.95rem, 2.2vw, 2.25rem)` + `line-height: 1.1`），减少跨页切换时的“信息权重跳变”。

表单区做了像素级统一：三页输入框统一到 `min-height: 3rem`、统一内边距与字号（`0.86rem/0.96rem + 1rem`），标签统一字重与微字距，标签前后间距通过 `margin-top` 规则收敛，错误提示行高/字号也一致化。按钮统一到 `min-height: 3rem`、同一字重与红紫主题阴影曲线，hover/active/focus 的反馈逻辑保持一致，保证交互节奏连贯。

页脚链接与辅助行（remember/forgot）同步做了对齐与节奏修正。页脚统一为可换行居中布局（含移动端），链接最小点击高度统一，避免三页出现“有的左对齐有的散开”的视觉割裂。登录、注册、忘记密码三页继续维持红紫渐变背景 + 白色表单卡片逻辑，且未引入新色系，避免主题冲突。

实现层面仅改 UI：`apps/web/src/app.css` 负责全局 auth 规则收敛，`apps/web/src/routes/auth/login/+page.svelte` 仅做标题标签层级调整；脚本内的校验、跳转、toast、mock 同步行为保持原样。构建验证通过：`corepack pnpm check`（0 error, 0 warning）与 `corepack pnpm build` 均成功。
