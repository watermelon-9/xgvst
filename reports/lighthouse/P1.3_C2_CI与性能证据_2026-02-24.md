## 3) 本地实测证据（移动端波动）

### 3.1 本次 3-run 结果（同一 URL）
执行：

```bash
bash scripts/lh-median-check.sh https://xgvst.com/ 97 3 reports/lighthouse/P1.3_C2_CI
```

证据文件：
- `reports/lighthouse/P1.3_C2_CI/lh-median-20260224-001407-run1.json`
- `reports/lighthouse/P1.3_C2_CI/lh-median-20260224-001407-run2.json`
- `reports/lighthouse/P1.3_C2_CI/lh-median-20260224-001407-run3.json`
- `reports/lighthouse/P1.3_C2_CI/lh-median-20260224-001407-summary.json`

结果：
- run1 = **65**
- run2 = **87**
- run3 = **87**
- median = **87** → 对阈值 97 判定 **FAIL**

### 3.2 波动根因（含 `cdn-cgi jsd`）
基于 `mobile-variance-evidence.json` 提取：

- run1（65分）
  - TBT: **41157ms**
  - `cdn-cgi/challenge-platform/scripts/jsd/main.js` bootup scripting: **25600ms**
- run2（87分）
  - TBT: **495ms**
  - 同脚本 bootup scripting: **1088ms**
- run3（87分）
  - TBT: **482ms**
  - 同脚本 bootup scripting: **1060ms**

**判定：移动端分数波动核心由 Cloudflare `cdn-cgi jsd` 注入脚本的主线程执行时间离散导致，直接拉高 TBT，进而拖低 performance 分数。**

## 4) 规避策略（针对移动端波动）

1. **CI 采用 3 次采样 + 中位数门禁**（已落地）
   - 避免单次极端抖动导致误判；同时保留低性能的失败约束。

2. **固定同区域 Runner / 固定网络出口**
   - 降低跨区域 Cloudflare 风控差异引发的挑战脚本波动。

3. **在性能测试白名单中排除挑战注入（建议）**
   - 对 Lighthouse 探针 UA/IP 做安全白名单或 challenge bypass（仅限性能测试路径），减少 `cdn-cgi jsd` 干扰。

4. **把 TBT 与 third-party script 开销纳入监控**
   - CI 报告中增加 `bootup-time` 对 `cdn-cgi` URL 的占比追踪，作为回归预警。
