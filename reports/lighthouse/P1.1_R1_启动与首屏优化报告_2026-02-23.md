# P1.1-R1 启动与首屏优化报告（Sub-B / Frontend）

日期：2026-02-23  
仓库：`/Users/mac/.openclaw/workspace/xgvst`

## 1) 变更目标与约束
- 目标：优化 `corepack pnpm --filter web dev` 启动 ready 时间，尽量压到 `<800ms`。
- 目标：优化首页首屏渲染，并给出 Lighthouse 本地 URL 对比。
- 约束：不引入重型库；不破坏现有路由与 API 抽象。

## 2) 实施改动

### 2.1 启动链路优化
1. **移除 UnoCSS 在 dev 链路中的运行时插件开销**
   - `apps/web/vite.config.ts`：移除 `UnoCSS()` 插件，仅保留 `sveltekit()`。
2. **改为轻量静态样式**
   - 新增 `apps/web/src/app.css`，以语义化 class 覆盖首页/市场页/详情页基础样式。
   - `apps/web/src/routes/+layout.svelte`：改为引入 `../app.css`。

### 2.2 首屏渲染优化
1. **首页结构轻量化**
   - `apps/web/src/routes/+page.svelte`：语义化卡片结构，减少依赖型样式解析路径。
2. **避免 dev 环境默认阻塞式 MSW 初始化**
   - `apps/web/src/hooks.client.ts`：仅当 `PUBLIC_ENABLE_MSW === 'true'` 时启动 MSW；并通过 `queueMicrotask` 延后初始化，避免影响首屏关键路径。
3. **保持路由与 API 抽象不变**
   - 保留 `/`、`/market`、`/detail/[symbol]` 路由。
   - 保留 `fetchUniverse()` 及 mock API 结构。

## 5) 风险与回滚

### 风险
1. **UnoCSS 功能退场风险**：后续若新增大量 utility class，需同步补充 `app.css`。
2. **MSW 默认关闭风险**：若开发者依赖 mock worker，需要显式设置 `PUBLIC_ENABLE_MSW=true`。

### 回滚
- 如需快速回滚：
  1) 恢复 `vite.config.ts` 中 `UnoCSS()`；
  2) `+layout.svelte` 改回 `import 'uno.css'`；
  3) `hooks.client.ts` 恢复原始 dev 默认启用逻辑。
