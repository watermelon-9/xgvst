# R1.3 首页与登录面板配色/层次对齐（v2.039语义）

## 1) 改造范围与约束对齐
本次仅做 **R1.3 最小改造**，不改动信息架构与认证闭环语义：
- 保持票 2.5 基线：`/auth/*` 页面继续无主导航栏；认证入口仍为邮箱账号体系（无手机号/微信/三方）。
- 保持现有样式体系：继续在 `apps/web/src/app.css` 内按现有 class 体系扩展；未引入新样式系统；未使用 inline style。
- 参考 v2.039 色彩基准：主轴使用红紫梯度 `#ff4e5f -> #9b4dff`，并映射到 body/hero/按钮与登录面板层次。

## 2) 首页（v3）关键区域配色与层次调整
已完成首页关键区域向 v2.039 视觉语义靠拢：
- **主背景（body）**：由单层冷紫线性背景，改为“红紫 radial + 深色底线性”叠层，主色锚点对齐 `#ff4e5f/#9b4dff`。
- **toolbar/hero（含顶部主导航条）**：统一提升为红紫线性梯度主视觉，增加高光边框与阴影，强化首屏品牌层。
- **核心按钮态**：导航 active、header 操作按钮 hover、首页分时按钮 active 等，统一采用红紫过渡高亮态，减少灰白按钮割裂感。
- **卡片层级**：三栏 pane、行情行项、脉冲卡片、K 线容器改为深底+微高光+边框层次，active/hover 引入红紫过渡，形成“背景 < 卡片 < 激活行”三级关系。

涉及文件：
- `apps/web/src/app.css`

## 3) 登录页面板（红紫基调 + 左右层次）
登录页在不改路由与认证逻辑前提下，完成面板层次贴近 v2 语义：
- **页面背景**：强化红紫 radial 氛围与深色底，和首页主色连续。
- **登录卡片**：新增 `auth-card-login`（仅登录页使用）并采用双列栅格：
  - 左侧为品牌/说明区（带分隔线，承担语义引导）；
  - 右侧为表单操作区（主交互焦点）。
- **视觉层次**：登录卡片增加叠层渐变与投影，卡片内部添加柔和红紫 overlay，形成“左说明、右操作”的 v2 类语义。
- **移动端**：在 `max-width: 820px` 自动回落单列，保持触控可读性与闭环流程不变。

涉及文件：
- `apps/web/src/routes/auth/login/+page.svelte`
- `apps/web/src/app.css`

## 4) 色值对照（改前/改后）与区域映射
| 区域 | 改前（主要色值） | 改后（主要色值） | 映射说明 |
|---|---|---|---|
| Body 主背景 | `linear`：`var(--xg-color-bg-purple)` 混合深紫 | `radial(#ff4e5f)` + `radial(#9b4dff)` + 深底 `#110b22` | 对齐 v2.039 的红紫主轴与空间感 |
| 首页 Hero/Toolbar | `rgb(92 24 133 / 70%) -> rgb(44 16 98 / 64%)` | `linear(108deg, #ff4e5f -> #9b4dff)` | 采用基准梯度，提升识别度 |
| 顶部主导航条 | `rgb(160 20 86 / 74%) -> rgb(90 28 186 / 72%)` | `linear(106deg, rgba(255,78,95,.88) -> rgba(155,77,255,.86))` | 与首页 hero 语义统一 |
| 首页卡片/行项 | 深紫半透明（静态） | 深底渐变 + 高光边 + active 红紫渐变 | 建立常态/激活态层级 |

| 登录面板 | 单列卡片 | `auth-card-login` 双列（左说明/右表单）+ overlay | 贴近 v2 左右层次语义 |
