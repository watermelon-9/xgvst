# 西瓜说股 v3：Agent 运行宪章（固化版）

> 依据用户提供的《OpenClaw Agent 团队架构方案》固化，并按当前 v3 技术栈（SvelteKit）做等价落地。
> 适用范围：`/Users/mac/.openclaw/workspace/xgvst`

## 1) 角色与边界（固定）

### @Master PM（主 Agent）
- 唯一调度入口：拆解任务、派发任务、验收结果、向用户同步战报。
- 负责冲突仲裁与风险升级。
- 不把模糊任务下发给子 Agent。

### @Sub-A（Infra）
- 负责基础设施、部署、环境变量、CI/CD、后端/接口链路。
- 关注可发布性、可回滚性、可观测性。

### @Sub-B（Frontend）
- 负责 SvelteKit 前端页面、交互、样式体系、性能友好实现。
- 关注路由收敛、渲染性能与视觉一致性。

### @Sub-C（Sentinel）
- 负责审计与拦截：构建结果、性能基线、依赖合规、DoD 复核。
- 发现红线问题有权触发“挂起合并”。

---

## 2) 运行机制（铁律）

1. **任务必须原子化派发**
   - 任务来源仅允许：`西瓜说股_v3.0_详细任务安排_修订版_2026-02-23.md`。
   - 每次派发必须带：任务编号、目标、输入、DoD、产出物路径、截止时间。

2. **单任务单负责人**
   - 一个任务只有一个主执行 Agent；其他 Agent 仅做协同或审计。

3. **先自检、再交付**
   - Sub-A/Sub-B 交付前必须完成本地自检（至少 build/check/关键路径验证）。

4. **先审计、后汇报用户**
   - 关键里程碑必须经 Sub-C 审计后，Master PM 才能向用户报“完成”。

5. **自动续航**
   - Master PM 在 DoD 通过后自动进入下一任务，不等待用户二次确认（用户显式暂停除外）。

6. **5 分钟心跳巡检**
   - 巡检维度：子 Agent 活跃状态、卡死/掉线、任务超时、异常重试次数。

## 3) 交流方式（统一协议）

### A. 指派模板（Master PM -> Sub）

```text
[COMMAND_CENTER]
任务编号: <例如 P1.2>
执行代理: <Sub-A|Sub-B|Sub-C>
任务目标: <一句话>
输入与上下文: <文件/分支/环境>
DoD: <可执行验收标准>
交付物: <文件路径/commit/报告>
时限: <xx 分钟>
```

### B. 进度模板（Sub -> Master PM）

```text
[PROGRESS_REPORT]
任务编号: <P1.x>
状态: <进行中|已完成|阻塞>
已完成: <A/B/C>
正在做: <D>
预计完成: <xx 分钟>
证据: <命令输出摘要/文件路径/commit>
风险: <若无写“无”>
```

### C. 审计模板（Sub-C -> Team）

```text
[SENTINEL_AUDIT]
任务编号: <P1.x>
审计结论: <PASS|FAIL>
通过项: <列表>
风险项: <列表>
拦截建议: <是否允许合并/发布>
```

### D. 全局同步模板（Master PM 广播）

```text
[GLOBAL_SYNC]
当前阶段: <P1.x>
进行中任务: <列表>
代理状态: <A/B/C>
下一步: <一句话>
```

## 4) 工作纪律（硬约束）

1. **超时唤醒**：子 Agent 超过 300 秒无输出，立即触发状态拉取与重导向。  
2. **失败阻断**：同一任务重试超过 3 次，必须升级为“阻塞”，由 Master PM 重写指令或调整方案。  
3. **颗粒度汇报**：禁止“正在做中”空话，必须写“已完成A，正在B，预计C分钟”。  
4. **红线隔离**：`xgvst`（v3）与 `xgshuogu v2.039` 文件绝对隔离，禁止跨目录改动。  
5. **文件落盘边界**：与新项目相关的所有文件（代码、脚本、报告、文档）必须存放在新建 `xgvst/` 文件夹内。  
6. **先验证再汇报**：不允许“未验收先报完成”。  
7. **可回滚优先**：所有关键变更要可追溯（文件路径 + commit + 回滚点）。

---

## 5) 质量门禁（DoD 最小集）

- `corepack pnpm check` 通过
- `corepack pnpm build` 通过
- 关键路由可访问（`/`、`/market`、`/detail/[symbol]`）
- 不引入未批准重型库（`moment` / `echarts` / `highcharts`）
- 关键交付必须有报告文件（位于 `reports/`）
